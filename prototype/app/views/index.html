Communicating funding prototype

A prototype to experiment in as part of the Communicating Funding alpha and beta phases. During this time, wee have tested and iterated five pages which deliver funding information to providers using data driven design patterns.



Beta PI16: calculations with factors and operations

Version Date Status Reasoning Actions
Version 6.1 1 July 2026 pending

Calculations and the 16 to 19 statement

In UR1, we will be testing three versions of a 16 to 19 allocation statement that includes varying levels of calculations and structures. This includes:

  • Minimal grouped: this includes the ‘first’ level of calculations for each funding line. We use the funding line name for the table headings.
  • Detailed grouped: this includes the ‘first’ and ‘second’ level of calculations for each funding line. We use both the funding line name and headings from the pdf for the table headings.
  • Ungrouped: this is based on the assumption that calculations can be shown in a logical order. It will follow the same approach as the calculation factor MVP where each factor listed also refers to the funding line.

Beta PI15: funding index journey, access to guidance & calculation factors

Building on PI13 & PI14, where we introduced a data‑driven, consolidated funding experience, we needed to ensure this new experience can co‑exist alongside the existing experience. While it improves access to more connected and reliable funding data, users still rely on the current system for historical data, audits, and wider financial processes.

The allocation statement has also been iterated now displaying calculation factors and statement version history, helping users better understand how their allocation has been calculated and track what has changed over time.

Calculation factors MVP and a holistic funding journey

Due to technical constraints, our MVP for this PI follows an unstructured approach. Calculation factors will show in the statement if they are:

  • given an external calculation factor name in CFS by Funding Operations.
  • not a value of 0 or null

In summary, our MVP for this new version of the allocation statement includes:

  • statement history, allowing users to view older versions of the statement
  • calculation factors
  • improved access to guidance

The minimum valuable product for the data-driven allocation statement is informed by 3 rounds of user testing across alpha and private beta.

  • designed as a pattern to be scaled out across Funding Service grants and funding streams, and used across the DfE to communicate funding information to end users

Version 3.1 16 October 2025 Tested

In this version, we have:

  • removed period initials (e.g. AS, AY, FY, etc.) from the statement heading
